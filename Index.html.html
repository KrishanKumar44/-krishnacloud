Linux

Q2. What is an Inode in Linux?

Answer

An Inode (Index Node) is a data structure on a Linux file system that stores all metadata about a file except its name and actual content. It contains info like file size, permissions, owner, and block locations.

Command

ls -i filename

Real Time Example

When troubleshooting a server that says "No space left on device" but `df -h` shows plenty of disk space, it usually means the system has run out of available Inodes due to millions of tiny files (like session files or mail queues).

Docker

Q3. Why are Docker Images immutable and how do layers work?

Answer

Docker images are built using a series of read-only layers. Each instruction in a Dockerfile creates a new layer. When you run a container, Docker adds a thin writable layer (Container Layer) on top.

Command

docker history image_name

Real Time Example

To optimize pipeline speeds and reduce disk usage, we place instructions that change frequently (like copying source code) at the bottom of the Dockerfile, and stable steps (like installing system packages) at the top to make full use of layer caching.

DevOps

Q4. What is Idempotency in Ansible?

Answer

Idempotency means that running an Ansible playbook multiple times on a target system will yield the same result, and it will only make changes if the system is not already in the desired state.

Command

ansible-playbook site.yml --check

Real Time Example

If you run a playbook to install Nginx on 50 servers, Ansible will check each server first. If 40 servers already have Nginx properly installed, it will skip them ("ok") and only install it on the remaining 10 ("changed"), ensuring no broken configuration or unnecessary service restarts occur.
